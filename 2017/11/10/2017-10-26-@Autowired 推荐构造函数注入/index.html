深究推荐 @Autowired 对构造函数注解的原因

发表于 2017-11-10 | 更新于: 2018-01-15 | 分类于 Spring |

最近在开发的时候，使用 @Autowired 注解 Field，发现 IDE 报了一个警告，如下：

翻译后的大概意思就是：
Spring 团队建议：总是在您的 bean 中使用构造函数建立依赖注入。总是使用断言强制依赖。

原来的写法是：

public class A {
/**

 *    注意多添加了一个 final 修饰
 */
private final B b；

@Autowired
public A(B b) {
    this.b = b;
}
...

}

@Getter
@Setter
@Service
public class B {

private String name;

private int age;

public User(){}

public User(String name, int age) {
    this.name = name;
    this.age = age;
}

@Override
public String toString() {
    return "Name=" + name + ", Age=" + age;
 }

}

public class A {
private String name;

@Autowired
private B b;

public A() {
    this.name = b.getName();    
}
...

}
```

能正常完成编译，但是运行时会抛出 NPE。

被 @Autowired 注解的 Field 的底层注入流程可以理解为在配置文件中配置 bean，然后使用 setter 注入。
而被 @Autowired 注解的 Constructor 的底层注入流程就相当于是使用构造函数进行依赖注入了。
